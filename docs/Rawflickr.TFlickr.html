type TFlickr = class(TObject)

Description

Base service class

This class is the basic "service info" wrapper used as Owner in TRESTApi and descendants.

Use this class only if you need single, manual instances of TRESTApi descendants; otherwise it'll we easier for you to instantiate a TFlickrEx object and access the API through its properties.

property ApiKey: String read FApiKey write SetApiKey;

API key associated to the application.

Actually, setting it does no more than setting a private field used by TRESTApi.SimpleCall and TRESTApi.SignedCall, but as it has other implications this will probably change in the future.

To access Flickr you must use a valid key issued by them (along with the associated shared secret, see Secret). For 23, though, you can use any random string (they suggest your application name) or the same key as for Flickr (which I don't recommend).

property Secret: String read FSecret write SetSecret;

The shared secret associated with your API key.

Actually, setting it does no more than setting a private field used by TRESTApi.SignedCall to build the request's signature, but as it has other implications this will probably change in the future.

Note that 23 doesn't issues API keys and thus they neither need or use a shared secret to sign calls.

property Service: TWebService read GetService;

Type of service we'are going to use.

This property determines the URI of the API endpoint and how signed calls (see TRESTApi.SignedCall) are signed.

Setting Service.ServiceType will also change Service.BaseURI to the corresponding KnownBaseURI; if needed, you can later set a different Service.BaseURI p.e. if you set: Service.ServiceType := stOther.

By itself, Service.ServiceType determines how calls are signed. Services other than Flickr —p.e. 23— issue neither API keys or secrets and, thus, don't need the MD5 signatures. Since sending a signature when it's not needed represents a security risk, we don't do it.

2007-01-07: This property has been completely reimplemented; if you have code like the old recommended: 

     flickr.Service := st23;
     UseEndPoint(ep23);
     flickr.ApiKey := 'My23ApiKey';

you should change it to something like: 

     flickr.Service.ServiceType := st23;
     flickr.ApiKey := 'My23ApiKey';

property Token: String read FToken write FToken;

Authentication token.

The value of this property is updated every time you call TAuth.checkToken, TAuth.getToken or TAuth.getFullToken. You can set it directly if you have a valid token from a previous authentication but in that case you may better use TAuth.checkToken; that way you won't only ensure that the token is still valid, but also update TFlickr.User with the correct user's info.

property Level: String read FLevel write FLevel;

Permissions level for which the token–if any–is valid.

It will be updated by a successful call to any of the TAuth methods that either use it or get it from its response (Note that TAuth methods always parse its responses.)
