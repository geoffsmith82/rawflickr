type TAuth = class(TRESTApi)

Description

Implements flickr.auth.* methods.

The authentication process for Flickr is explained in the document "Authentication API" <http://www.flickr.com/services/api/auth.spec.html>. For 23, consult "User authentication" at <http://www.23hq.com/doc/api/auth>

Methods

procedure ParseResponse(resp: String); override;

Parses checkToken / getToken RSP and (if sucessful) updates TFlickr(Owner).UserInfo, Token and Level (i.e. the account info).

function ParseFrobResp(resp: String): String;

Parses a getFrob response and returns (if sucessful) a frob.

function checkToken(Token: String): String;

Implements flickr.auth.checkToken As a collateral effect it updates the Owner's account info i.e. UserInfo, Token and Level (see TFlickr)

function getFrob: String;

Implements flickr.auth.getFrob. NOTE that this call doesn't return any XML payload. It will instead try to parse the response and return the frob; If it doesn't succeeds you'll receive an exception raised from the default parser.

function getFullToken(miniToken: String): String;

Implements flickr.auth.getFullToken. Same collateral effects that checkToken

function getToken(frob: String): String;

Implements flickr.auth.getToken. Same collateral effects that checkToken

function GetLoginLink(perms: String; frob: String): String;

Returns an AnsiString containing the unencoded URI to which the user must navigate in order to authenticate the application.

function GetLoginLinkW(perms: String; frob: String): WideString;

Returns a WideString containing the unencoded URI to which the user must navigate in order to authenticate the application.

constructor Create(Owner: TFlickr);

Class constructor. Unlike the inherited one, it doesn't accept a TSignOption parameter. Since all flickr.auth.* methods must be signed, I introduced this constraint to circumvent an already corrected bug in Flickr and prevent a common error.
