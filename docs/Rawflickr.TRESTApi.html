type TRESTApi = class(TObject)

Description

Base REST API Implementation.

This generic, base class implements most of the behaviour needed to access a web service. It was built specifically to access Flickr(tm) but you may probably descend from it the classes needed for services using a like API paradigm.

Even though it isn't an abstract one, you shouldn't need to make instances of this class; rather, derive a child class implementing the specific behaviour you need, like this library does.

Prepares and issues an unsigned call to the endpoint unless SignAll = True, in which case control will be transferred to SignedCall.

procedure ParseResponse(resp: String); dynamic;

Default response parser. All it does is to build a INI-like structure using a simple XMLToStrings function

procedure CheckError(XMLResp: String);

REST response checking.

This method checks whether a REST response status is "fail", "ok" or isn't valid—i.e. the status is neither of those two or it isn't even a valid XML REST payload.

Exceptions raised
EFlickrError
descendant if the status is "fail".
EFlickrBadResponse
if the status is neither "fail" or "ok", case that includes invalid REST payloads.
procedure CheckAndSave(XMLResp: String; Stream: TStream); overload;

Saves a response into a stream.

Since the response is saved after checking it with CheckError and CheckError raises an exception on error conditions, you can only save valid, non-fail responses with this procedure.

procedure CheckAndSave(XMLResp, FileName: String); overload;

Saves a response into a file.

All this method does is to create a file stream and then call the "stream" variant. See its notes

function FlickrCall(const method: String; Signed: Boolean = False): String; virtual;

Generic Flickr method call

All it does is select and call either SimpleCall or SignedCall, depending of the value of the parameter Signed

property SignAll: Boolean read FSignAll write FSignAll default False;

Tells TRESTApi when to issue signed call

When set to True TRESTApi will issue all calls as signed, even if made through SimpleCall.

property Request: TWebParams read FRequest write FRequest;

Request parameters for a call to flickr.

Use this property to set the parameters of the request if you're implementing a flickr method in a TRESTApi descendant. Other users of the library should consider this value as read-only; after a flickr method call you'll find here all the parameters as passed to the HTTP client.

property LastResponse: String read FResponse;

Latest RSP received from the web service. Read-only.

property Owner: TFlickr read FOwner;

TFlickr or descendant "owning" this class' instance.

Though this "ownership" is a somewhat difussed concept, all descendants of this class get fields from its Owner, so you need to provide them a valid instance of TFlickr–of course, unless the object is indeed a TFlickr instance's property.

property Throttle: Cardinal read FThrottle write FThrottle default 10;

Controls a delay between calls.

If non-zero, any call to ??? will wait for 'value' 100th of seconds before returning, thus implementing a crude "bandwidth" control.

property RandomWait: Boolean read FRandomWait write FRandomWait;

Controls how Throttle will be interpreted.

If set, Throttle will be interpreted as the maximum wait value, rather than as an uniform wait value.
